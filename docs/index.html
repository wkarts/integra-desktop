Integra Desktop

Portal técnico de release

Aplicação desktop em Tauri para conversão NFS-e/NFe em arquivos de importação Prosoft, com fallback legado e pipeline completa de CI/CD no GitHub.

Entrega contínua

Push em main dispara validação. Quando houver commit compatível com Conventional Commits, a release é versionada com SemVer e os binários são publicados no GitHub Releases.

Empacotamento

Windows, Linux e macOS são tratados pela pipeline de release usando Tauri Build. O versionamento do projeto fica sincronizado em VERSION, package.json, Cargo.toml e tauri.conf.json.

Segredos necessários

Para assinatura e updater, configure as chaves do Tauri e, quando necessário, os certificados de Windows/macOS no repositório.

Fallback legado

O módulo legado continua incorporado ao projeto e permanece disponível como rota separada, sem contaminar o core novo em Rust.

Arquivos operacionais

  • .github/workflows/ci.yml — validação de PR e push
  • .github/workflows/release.yml — versionamento, release e binários
  • .github/workflows/pages.yml — publicação desta documentação
  • release.config.mjs — automação semantic-release
  • scripts/release/* — sincronismo de versão e metadados de release
  • scripts/ci/* — validação de título de PR e sincronismo de versões
